InsecureShop — Web App Assessment

Project Overview

This is a medium difficulty web application assessment (lab environment). Purpose: practise common web vulnerabilities and produce a short professional report.

Scope

  • Target: InsecureShop (lab VM)
  • Focus: Authentication, input validation, and data leakage
  • Out of scope: Denial of Service, social engineering

Tools Used

Nmap, Burp Suite (community), sqlmap (selective), curl, browser devtools, small Python scripts

Steps — Methodology

  1. Recon: Nmap port scan and service enumeration.
  2. Web discovery: map endpoints and parameters (Burp Spider / manual).
  3. Auth tests: brute-force weak credentials (manual / wordlist) on login page.
  4. Input validation: test forms for SQLi, XSS; use Burp to intercept and modify requests.
  5. Post-exploitation: extract data from exploited endpoints and document impact.

Findings & Proof

1) SQL Injection (medium) — Found in search parameter `q` on /search page. Exploitation allowed extraction of product IDs and usernames (read-only).

Example payload (manual): ' OR '1'='1' --

2) Stored XSS (low) — Feedback form stored unsanitized input; proof: inserted <script>alert(1)</script> triggered in admin view.

3) Weak credentials: found default admin/admin accessible on /admin — low impact if credentials are default.

Remediation Recommendations

  • Parameterized queries / prepared statements for DB access.
  • Sanitize and escape user input; apply CSP headers to reduce XSS risk.
  • Enforce strong credentials and rate-limit login attempts.
  • Log and monitor admin access; remove default credentials.

Lessons Learned

This lab taught a realistic workflow: reconnaissance → mapping → focused exploitation → remediation recommendations. Using Burp Suite for request manipulation and small Python scripts for automating data extraction sped the process.
